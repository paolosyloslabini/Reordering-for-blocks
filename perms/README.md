# perms/

Generated permutation files organized by algorithm.

## Structure

```
perms/
├── graphblas/           # GraphBLAS-generated permutations
│   ├── random1D/
│   │   ├── matrix1.perm
│   │   └── matrix2.perm
│   └── random2D/
│       └── ...
├── SB_rcm/              # SparseBase RCM permutations
├── SB_degree/           # SparseBase degree ordering
├── SB_gray/             # SparseBase Gray code
├── SB_amd/              # SparseBase AMD
├── SB_metis/            # SparseBase METIS
├── SB_rabbit/           # SparseBase Rabbit
├── SB_slashburn/        # SparseBase SlashBurn
└── SPARTA_reorder/      # SPARTA blocking permutations
```

## Permutation File Format (.perm)

**Single line** - Symmetric or row-only permutation:
```
3 1 4 2 5
```

**Two lines** - Asymmetric permutation (row permutation, then column permutation):
```
3 1 4 2 5
2 4 1 5 3
```

**Important:** Indices are **1-based** (first element is index 1, not 0).

## Naming Convention

Permutation files are named after their source matrix:
```
{matrix_name}.perm
```

Example: `af_shell1.perm` for matrix `af_shell1.mtx`

## Generation

Permutations are generated by tools in `MtxPerm/`:

| Directory | Tool |
|-----------|------|
| `graphblas/random*` | `MtxPerm/RANDOM/random_perm.py` |
| `SB_*` | `MtxPerm/SPARSEBASE/reorder` |
| `SPARTA_reorder` | `MtxPerm/SPARTA/sparta_reorder` |

Configured via [yamls/perms.yaml](../yamls/perms.yaml).

## Usage in Operators

```python
# In operators, permutations are loaded via:
from operators.config import load_permutation

row_perm, col_perm = load_permutation("perms/SB_rcm/matrix.perm")
# row_perm, col_perm are 0-based numpy arrays after loading
```

## Identity Permutation

The file `identity_perm.txt` at repository root is used as a no-op baseline:
```
# Just the sequence 1 2 3 ... n
```
